# לוח בקרה ניהולי — נכסי אירוח באירופה (MVP)

לוח בקרה (Dashboard) בעברית, RTL מלא, לחברת נדל"ן ישראלית המנהלת מלונות,
פנסיונים ודירות נופש באירופה. המערכת מציגה רווח והפסד, הכנסות, הוצאות,
תפוסה, הזמנות, תחזוקה וחריגות — מתוך **קובץ Excel שאתה מעלה לאתר**.

**מודל העבודה:** בכל פעם שאתה מעלה את קובץ ה-Excel, הנתונים מתעדכנים
לפי הקובץ. אין שרת, אין חיבור חי, אין התקנות מצד Microsoft — פשוט מעלים
קובץ והכל מתחשב מחדש. הקובץ שהועלה נשמר בדפדפן, כך שאחרי רענון הדף
הנתונים נשארים עד שמעלים קובץ חדש.

> **שלב MVP — לצפייה בלבד.** אין עריכה ואין שינוי של הקובץ. המערכת
> מתייחסת לקובץ ה-Excel כמקור קריאה-בלבד (read-only).

---

## 1. מה נבנה

אפליקציית **React + TypeScript + Vite + Tailwind CSS**, עם תרשימים ב-**Recharts**.
תשע מסכים, תפריט צד בעברית, פס פילטרים גלובלי, וכפתור **"רענון נתונים"**.

המסכים:

1. **דאשבורד ראשי** — כרטיסי KPI, הכנסות מול הוצאות, רווח/הפסד לפי נכס,
   פילוח לפי פלטפורמה, פילוח הוצאות, נכס מוביל/חלש, וטבלת סיכום כל הנכסים.
2. **נכסים** — כרטיס לכל נכס, טבלת חדרים, וטבלת עמלות לפי פלטפורמה.
3. **רווח והפסד (P&L)** — דוח מפורט (שורות = סעיפים, עמודות = נכסים) + שינוי חודשי.
4. **הכנסות** — לפי נכס, חודש, פלטפורמה, וסוג הכנסה (הזמנות / הכנסות נוספות).
5. **הוצאות** — לפי קטגוריה, סטטוס תשלום, אמצעי תשלום; סכום מלא / ששולם / נותר.
6. **תפוסה** — לפי נכס, חודש וחדר; לילות תפוסים מול לילות זמינים.
7. **הזמנות** — כל ההזמנות מגיליון occupation, עם סינון לפי סטטוס.
8. **תחזוקה** — דיווחי תקלות, עלויות וסטטוסי טיפול.
9. **חריגות בנתונים** — התראות על הפסדים, הוצאות/הכנסות חריגות, נתונים חסרים ועוד.

## 2. איך להריץ

```bash
npm install
npm run dev
```

הדפדפן ייפתח אוטומטית ב-`http://localhost:5173`. לבנייה לפרודקשן:
`npm run build` ואז `npm run preview`.

### איך מעלים את הקובץ ומעדכנים נתונים

יש שתי דרכים זהות להעלאה:

1. כפתור **"העלאת קובץ Excel"** הכחול בראש המסך (למעלה משמאל).
2. אזור הגרירה (Drag &amp; Drop) שמופיע במצב דמו — גוררים אליו את הקובץ.

ברגע שמעלים קובץ: כל הנתונים, המדדים, הגרפים והטבלאות מתחשבים מחדש,
החיווי הופך לירוק **"מעודכן מקובץ Excel"**, וזמן העדכון מתעדכן.
בכל העלאה חוזרת — הנתונים מתחלפים לחלוטין לפי הקובץ החדש.

> הקובץ שהועלה נשמר ב-`localStorage` של הדפדפן, כך שרענון הדף לא מאבד
> את הנתונים. כפתור **"נקה / דמו"** מנקה אותם וחוזר לתצוגת הדמו.

## 3. מבנה הפרויקט — הפרדה מלאה בין נתונים לתצוגה

```
src/
├─ data/                ← שכבת הנתונים (אין כאן UI)
│  ├─ types.ts            מבנה הנתונים הפנימי (תואם לגיליונות)
│  ├─ normalize.ts        טיפול בשמות עמודות/גיליונות עם שגיאות כתיב
│  ├─ mockData.ts         ⚠️ נתוני דמו בלבד (פיקטיביים)
│  ├─ loadExcel.ts        פירוק קובץ ה-Excel שהועלה (SheetJS)
│  └─ DataContext.tsx     המאגר המרכזי + רענון נתונים
├─ lib/                 ← שכבת החישובים (אין כאן UI)
│  ├─ calculations.ts     כל החישובים העסקיים
│  ├─ anomalies.ts        זיהוי חריגות
│  └─ format.ts           עיצוב יורו (€), אחוזים, תאריכים בעברית
├─ context/FilterContext.tsx   פילטרים גלובליים
├─ components/          ← רכיבי תצוגה (כרטיסים, טבלאות, גרפים, Header, Sidebar)
└─ pages/              ← תשעת המסכים
```

**עיקרון מנחה:** הקומפוננטות לעולם לא ניגשות ישירות לקובץ או ל-mockData —
הן קוראות רק מ-`DataContext`, וכל החישובים נמצאים ב-`lib/calculations.ts`.

## 4. איפה נמצאים נתוני הדמו

📁 **`src/data/mockData.ts`** — וזה **המקום היחיד** עם נתונים פיקטיביים.
כל הנתונים שם מסומנים בבירור כ-MOCK, נוצרו אקראית (seed קבוע), ותואמים
למבנה הגיליונות שלך (4 נכסים, ינואר–יוני 2026):

- Black Forest Hotel (BF) · Chemnitz Apartments (CH1) · Helvetia Tabiano (TAB1) · Tavernola Apartments (TAV)

כשמופעלים נתוני דמו מוצג באנר כתום בראש המסך וגם בתפריט הצד: **"⚠️ נתוני דמו פעילים"**.
לא הומצא ולא הושלם שום נתון מהקובץ האמיתי שלך.

## 5. איך מתבצעים החישובים

הכול מחושב **ישירות מגיליונות המקור** (occupation, Extras, Expenses, Maintence,
Rooms, Properties) — לא מהגיליונות המוכנים (Pivot / P&L) שב-Excel.

| מדד | חישוב |
|---|---|
| הכנסה כוללת | `TotalNetPrice` (occupation) + `Amount` (Extras) |
| הוצאה כוללת | `PaidAmount` (Expenses) + `Amount` (Maintence) |
| רווח נקי | הכנסה כוללת − הוצאה כוללת |
| שולי רווח | רווח נקי / הכנסה כוללת |
| תפוסה (הערכה) | לילות תפוסים / (מספר חדרים × ימים בתקופה) |
| עמלות | סכום `commission` מ-occupation (ומושווה לטבלת העמלות) |

דוח ה-P&L מציג הכנסות ברוטו לפי פלטפורמה, ועמלות כשורת הוצאה נפרדת —
שתי השיטות (ברוטו ונטו) נותנות אותו רווח נקי.

> **טיפול בשגיאות הכתיב שבקובץ** (ב-`normalize.ts`): המערכת מזהה גם
> `ProperyName`, `Commision %`, `Maintence`, `InvocieDate`, `dashbords`, ו-`occupation`
> באותיות קטנות — בלי לקרוס.

## 6. איך הקובץ נקרא ונשמר

כל הלוגיקה של קריאת הקובץ נמצאת ב-📁 **`src/data/loadExcel.ts`**:

- `loadFromArrayBuffer(buffer)` — מפרק קובץ xlsx אמיתי באמצעות הספרייה
  SheetJS, כולל **פיצול שתי הטבלאות בגיליון Properties** (A–C ו-E–H),
  וזיהוי גמיש של שמות הגיליונות (גם עם שגיאות הכתיב).
- `src/data/DataContext.tsx` — מקבל את הקובץ מההעלאה, מפרק אותו, מעדכן את
  כל המסכים, ושומר עותק ב-`localStorage` כדי שהנתונים יישארו אחרי רענון.

אין שרת ואין חיבור רשת — הכול קורה בדפדפן עצמו. זה גם אומר שהקובץ שלך
**לא עוזב את המחשב** ולא נשלח לאף שרת חיצוני.

## 7. רעיונות לשדרוג עתידי (לא חובה ל-MVP)

1. **העלאה לשרת/ענן** — אם בעתיד תרצה שכמה מנהלים יראו את אותם נתונים בלי
   שכל אחד יעלה קובץ, אפשר להוסיף שרת קטן שמאחסן את הקובץ האחרון.
2. **חיבור אוטומטי ל-OneDrive / Graph** — אם תחזור למודל "משיכה אוטומטית",
   הקוד בנוי כך שקל להוסיף מקור נתונים נוסף לצד ההעלאה.
3. **מסד נתונים והיסטוריה** — שמירת כל קובץ שהועלה כדי להשוות מגמות לאורך זמן.
4. **התחברות והרשאות** — כניסת משתמשים והרשאות צפייה/עריכה.

## 8. מה אצטרך ממך בהמשך

1. אישור על **לוגיקת החישוב** — במיוחד: האם "הכנסה כוללת" צריכה להיות נטו
   (כפי שמוגדר כעת) או ברוטו, והאם להכליל הוצאות שטרם שולמו.
2. שמות הסטטוסים המדויקים בעמודות Status (חדרים, הזמנות, תשלומים, תחזוקה) —
   כדי לצבוע אותם נכון.
3. המטבע: כרגע הכול מוצג ב-**יורו (€)**. אם חלק מהנכסים במטבע אחר — צריך
   להגדיר שער המרה.
4. אם מבנה הקובץ ישתנה בעתיד (עמודות/גיליונות חדשים) — נעדכן את שכבת
   הנורמליזציה בהתאם.
